❯ comando de docker para volver a crear de cero las imagenes incluyendo la      
  limpieza de cache                                                             

● Para recrear las imágenes de Docker desde cero eliminando cualquier rastro de
  caché (lo cual es muy recomendable después de cambiar dependencias o archivos
  de configuración), utiliza el siguiente comando:

  docker compose build --no-cache && docker compose up -d

  Explicación detallada de lo que hace:

  1. docker compose build --no-cache: Fuerza a Docker a ignorar todas las capas
  almacenadas en caché y ejecutar cada instrucción del Dockerfile desde el
  principio. Esto garantiza que se instalen las versiones más recientes de las
  librerías y se incluyan los cambios en el código.
  2. docker compose up -d: Levanta los contenedores en segundo plano (detached 
  mode), recreando los servicios que hayan cambiado.

  ---
  Si quieres una limpieza "total" (Borrar contenedores, redes e imágenes 
  antiguas)

  Si notas que el comando anterior no es suficiente o quieres limpiar el sistema
   por completo antes de empezar, ejecuta esta secuencia:

  # 1. Detiene y elimina contenedores, redes y volúmenes asociados al proyecto
  docker compose down -v

  # 2. Elimina todas las imágenes huérfanas y caché de construcción del sistema
  docker builder prune -a -f

  # 3. Construye desde cero y levanta
  docker compose up -d --build

  Recomendación: Para tu caso actual (después de corregir el error 422 y
  modificar esquemas), el primer comando (build --no-cache) es el más eficiente
  y suficiente.

● Este error no tiene nada que ver con tu código, es un problema de conectividad
   de red entre tu computadora y los servidores de Docker Hub.

  El mensaje i/o timeout al intentar hacer el lookup de registry-1.docker.io
  significa que tu Docker no puede resolver la dirección DNS para descargar la
  imagen de Python.

  Aquí tienes los pasos para solucionarlo, desde lo más simple a lo más
  avanzado:

  1. Verificar la conexión a internet

  Asegúrate de que tienes internet activo. Si estás usando una VPN o un Proxy,
  desactívalos temporalmente, ya que suelen bloquear las peticiones de Docker.

  2. Reiniciar el servicio de Docker (Solución más común)

  A veces el demonio de Docker pierde la configuración de red.
  - Haz clic derecho en el icono de Docker Desktop en la barra de tareas y
  selecciona Restart.
  - O reinicia la aplicación por completo.

  3. Cambiar el DNS de tu computadora (Solución definitiva)

  El error menciona que el servidor DNS 192.168.65.5 está fallando. Te
  recomiendo cambiar los DNS de tu tarjeta de red por los de Google, que son
  mucho más estables:
  - DNS Primario: 8.8.8.8
  - DNS Secundario: 8.8.4.4

  Cómo hacerlo en Windows:
  1. Panel de Control $\rightarrow$ Centro de redes y recursos compartidos
  $\rightarrow$ Cambiar configuración del adaptador.
  2. Clic derecho en tu conexión $\rightarrow$ Propiedades.
  3. Selecciona "Protocolo de Internet versión 4 (TCP/IPv4)" $\rightarrow$
  Propiedades.
  4. Selecciona "Usar las siguientes direcciones de servidor DNS" y escribe
  8.8.8.8 y 8.8.4.4.

  4. Forzar la descarga si el DNS ya funciona

  Una vez que hayas reiniciado Docker o cambiado el DNS, intenta ejecutar solo
  el pull de la imagen antes de construir:

  docker pull python:3.12-slim

  Si este comando termina con éxito, entonces ya puedes ejecutar el build:
  docker compose build --no-cache

  Resumen: Tu computadora no pudo "encontrar" el servidor de Docker en internet.
   Reinicia Docker Desktop y, si persiste, cambia tus DNS a los de Google.
