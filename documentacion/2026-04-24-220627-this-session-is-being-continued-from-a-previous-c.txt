⏺ Para cargar y procesar los triángulos, el core actuarial utiliza el Método 
  Chain Ladder (Escalera de Cadena). A continuación te detallo la secuencia
  matemática que ocurre en el código:

  1. Agregación de Datos (Capa SQL)

  Antes de llegar a Python, el sistema realiza una suma agrupada en la base de
  datos para reducir el volumen de datos:
  $$\text{Total}_{o, d} = \sum (\text{monto_pagado} + \text{monto_reserva})$$
  Donde:
  - $o$: Año de origen (Origin Year).
  - $d$: Año de desarrollo (Development Year).

  2. Construcción de la Matriz (Pivot)

  El motor transforma esos datos en una matriz donde las filas son los años de
  origen y las columnas los años de desarrollo. El resultado es un Triángulo de 
  Siniestros:

  ┌────────────┬────────────┬────────────┬────────────┬─────────┐
  │ Año Origen │   Dev 0    │   Dev 1    │   Dev 2    │   ...   │
  ├────────────┼────────────┼────────────┼────────────┼─────────┤
  │ 2021       │ $C_{21,0}$ │ $C_{21,1}$ │ $C_{21,2}$ │ ...     │
  ├────────────┼────────────┼────────────┼────────────┼─────────┤
  │ 2022       │ $C_{22,0}$ │ $C_{22,1}$ │ $\dots$    │ $\dots$ │
  ├────────────┼────────────┼────────────┼────────────┼─────────┤
  │ 2023       │ $C_{23,0}$ │ $\dots$    │ $\dots$    │ $\dots$ │
  └────────────┴────────────┴────────────┴────────────┴─────────┘

  3. Cálculo de Factores de Desarrollo (LDF)

  El sistema calcula los Loss Development Factors (LDF) basándose en la
  experiencia histórica. Es el cociente entre la suma de una columna y la
  anterior:

  $$\text{LDF}{i \to i+1} = \frac{\sum{j=1}^{n} C_{j, i+1}}{\sum_{j=1}^{n} C_{j,
   i}}$$

  - Ejemplo: Si en el año de desarrollo 0 se pagaron $100M y en el año 1 se
  pagaron $120M, el factor $\text{LDF}_{0 \to 1}$ es $1.2$ (un crecimiento del
  20%).

  4. Proyección del Siniestro Final (Ultimate Loss)

  Para cada año de origen, el motor toma la suma actual de lo pagado y lo
  multiplica por los factores restantes hasta llegar al final del periodo de
  desarrollo:

  $$\text{Ultimate Loss}o = (\sum{d=0}^{k} C_{o,d}) \times \prod_{i=k}^{N-1}
  \text{LDF}_{i \to i+1}$$

  Donde:
  - $k$: Es el último año de desarrollo con datos para ese año de origen.
  - $N$: Es la edad máxima de desarrollo del triángulo.

  5. Estimación del IBNR (Incurred But Not Reported)

  Finalmente, la reserva técnica actuarial (IBNR) es la diferencia entre la suma
   de todas las pérdidas proyectadas y lo que ya se ha pagado/reservado
  contablemente:

  $$\text{IBNR} = \sum (\text{Ultimate Losses}) - \sum (\text{Actual Losses})$$

  Resumen del Flujo en el Código:

  - build_triangle: Ejecuta la matriz pivot.
  - calculate_ibnr:
    a. Calcula los $\text{LDF}$ (paso 3).
    b. Aplica el multiplicador de severidad (si el usuario lo ajusta en la UI).
    c. Calcula el $\text{Ultimate Loss}$ para cada fila (paso 4).
    d. Resta la suma actual para dar el IBNR final (paso 5).
